Практическая работа: Полиморфизм (Проект Shapes)

## 🎯 Цель

Закрепить навыки использования наследования, переопределения методов и полиморфизма в Java.

## 📝 Описание задания

Необходимо разработать консольное приложение `Shapes`, моделирующее работу с различными геометрическими фигурами.

---

## ⚙️ Требования к реализации

### Базовая архитектура

* Создайте общий родительский класс для всех фигур.


* Реализуйте не менее трех различных фигур, наследующихся от родительского класса.


* Для каждой фигуры задайте необходимые параметры (например, радиус, ширина, высота).


* Реализуйте вычисление площади и вывод информации о фигуре.


* Используйте аннотацию `@Override` для переопределения методов, чтобы каждая фигура выполняла вычисления и вывод данных по-своему.



### Главный класс (App)

* В главном классе `App` создайте несколько объектов разных фигур.


* Продемонстрируйте полиморфизм, сохранив объекты разных типов в одной коллекции (или массиве) общего типа.


* Обработайте коллекцию в цикле, вызывая одинаковые методы для всех объектов без проверки их конкретного типа.


* Выведите результаты работы программы в консоль.



---

## 🚀 Дополнительное задание

Для более глубокого понимания принципов ООП реализуйте следующий функционал:

* Вычисление общей площади всех созданных фигур.


* Подсчет количества фигур каждого типа.


* Демонстрацию восходящего (upcasting) и нисходящего (downcasting) приведения типов.


* Использование оператора `instanceof` для проверки принадлежности объекта к определенному классу.



---

## 💡 Ожидаемый результат

Программа должна наглядно демонстрировать, что объекты разных классов могут обрабатываться через общий тип. При вызове общих методов автоматически используется реализация того соответствующего дочернего класса, которым объект фактически является.
